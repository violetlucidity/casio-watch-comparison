PHASE 3: IMPLEMENT TOUCH DRAG (Priority 3 - Nice to Have)

Goal: Enable drag-to-overlay comparison on mobile

Current Issue: Using mousedown/mousemove only works on desktop

Solution: Use pointer events (works for both mouse and touch)

Key Code Changes:

javascript
svg.addEventListener('pointerdown', (e) => {
    e.preventDefault();
    svg.dataset.isDragging = true;
    svg.dataset.startX = e.clientX - (svg.dataset.offsetX || 0);
    svg.dataset.startY = e.clientY - (svg.dataset.offsetY || 0);
});

document.addEventListener('pointermove', (e) => {
    const svg = document.querySelector('.watch-svg[data-is-dragging="true"]');
    if (!svg) return;
    const x = e.clientX - svg.dataset.startX;
    const y = e.clientY - svg.dataset.startY;
    svg.style.transform = `translate(${x}px, ${y}px)`;
    svg.dataset.offsetX = x;
    svg.dataset.offsetY = y;
});

document.addEventListener('pointerup', () => {
    document.querySelectorAll('.watch-svg').forEach(s => {
        s.dataset.isDragging = false;
    });
});

Success Criteria:

    Can drag SVG watches on mobile

    Smooth movement without jitter

    Works in both portrait and landscape

    Desktop mouse drag still works
